# Spacing

The spacing system is built on a **4px base unit**. All margins, paddings,
and gaps should reference this scale to maintain visual rhythm.

## Guidelines

| Context | Recommended spacing |
|---------|-------------------|
| Inside cards | `p-5` (20px) to `p-6` (24px) |
| Page margins | `p-6` (24px) to `p-8` (32px) |
| Between cards | `gap-4` (16px) to `gap-6` (24px) |
| Form field gap | `gap-4` (16px) |
| Table cell padding | `px-3 py-2` (12px / 8px) compact, `px-4 py-3` comfortable |
| Button internal | `px-4 py-2` default, `px-3 py-1.5` small |
| Section spacing | `space-y-6` (24px) to `space-y-8` (32px) |

---

## Principles

1. **Generous whitespace**: The system favours breathing room. When in doubt,
   add more space, not less.
2. **Consistent scale**: Always pick from the scale. Avoid arbitrary pixel values.
3. **Density control**: Data-heavy views may use compact spacing (`p-3`, `gap-3`);
   overview pages use comfortable spacing (`p-6`, `gap-6`).
