# Color System

The AFENDA design system uses a high-contrast design language:
deep blue primary, restrained neutral grays, functional status colors, and
a gold accent used sparingly. Semantic tokens adapt between light and dark mode.

---

## Brand blue scale

The primary palette. Use `500` for default actions, `700` for headers and emphasis,
lighter tints for hover states and backgrounds.

### TypeScript

```ts
import { moduleColors, getModuleColor, colors } from "@afenda/ui-core/tokens";

moduleColors.hr;              // "#1A9D6E"
getModuleColor("recruitment"); // "#E06528"
colors.brand[500];            // "#005EB8"
colors.status.success;        // "#2E7D32"
```

## Semantic Tokens

Semantic tokens adapt to light and dark mode via the `.dark` class on `<html>`.

| Token | Usage |
|-------|-------|
| `--background` | Page background |
| `--foreground` | Default text |
| `--card` / `--card-foreground` | Card surfaces |
| `--popover` / `--popover-foreground` | Dropdown/popover surfaces |
| `--primary` / `--primary-foreground` | Primary actions, links |
| `--secondary` / `--secondary-foreground` | Secondary buttons |
| `--muted` / `--muted-foreground` | Subtle backgrounds, helper text |
| `--accent` / `--accent-foreground` | Hover/focus highlights |
| `--destructive` / `--destructive-foreground` | Error, danger actions |
| `--success` / `--success-foreground` | Positive confirmations |
| `--warning` / `--warning-foreground` | Caution states |
| `--info` / `--info-foreground` | Informational highlights |
| `--border` | Default border colour |
| `--input` | Input border colour |
| `--ring` | Focus ring colour |

## Border Radius

Tightened for a crisper, more professional feel.

| Token | Value |
|-------|-------|
| `--radius` | `0.375rem` (6px) |
| Tailwind `rounded-lg` | `var(--radius)` |
| Tailwind `rounded-md` | `calc(var(--radius) - 2px)` |
| Tailwind `rounded-sm` | `calc(var(--radius) - 4px)` |
