## Foundation

The design system is built on a rigorous token layer:

| Layer | Description |
|-------|-------------|
| **Colors** | Brand blue scale (50–950), neutral grays, gold accent, status colors, 9 module colours |
| **Typography** | Source Serif 4 (headings) + Inter (body) — the serif/sans split signals authority for headlines and clarity for data |
| **Spacing** | 4px base unit, 16 stops from 0 to 96 |
| **Shadows** | 5-level elevation system (xs–xl) |
| **Motion** | 3 duration tiers + 4 easing curves, respects `prefers-reduced-motion` |

## Design Principles

1. **Serif for authority** — Use `font-display` (Source Serif 4) on page titles
   and KPI values. Never for body text.

2. **Action titles** — Page headings state conclusions, not topics.
   "Revenue grew 15% driven by enterprise" not "Revenue Overview".

3. **Functional color** — Color signals meaning (status, trend, module).
   Never decoration.

4. **Generous whitespace** — Despite high data density, components breathe.
   When in doubt, add space.

5. **60-second rule** — Every view must be grasped in 60 seconds or less.
