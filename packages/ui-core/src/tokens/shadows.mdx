# Shadows & Elevation

Shadows communicate depth and hierarchy. The system provides five levels,
each mapped to a specific UI context. Use restraint — most surfaces
need `shadow-sm` or no shadow at all.

---

## Usage

```html
<div class="shadow-sm rounded-lg bg-card">Card content</div>
<div class="shadow-md rounded-lg bg-popover">Dropdown panel</div>
<div class="shadow-lg rounded-lg bg-card">Dialog content</div>
```

---

## Principles

1. **Less is more**: Most surfaces need `shadow-sm` or no shadow. Reserve
   larger shadows for elements that truly float above the page.
2. **Consistent mapping**: Popovers always use `shadow-md`, dialogs always
   use `shadow-lg`. Don't mix levels for the same component type.
3. **Dark mode**: Shadows become less visible on dark backgrounds.
   Dark mode relies more on border contrast than shadow depth.
