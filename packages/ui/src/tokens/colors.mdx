# Color System

The AFENDA design system uses CSS custom properties for all colours.
Semantic tokens change between light and dark mode automatically.
Module colours remain constant across themes.

---

## Module Colours

Each AFENDA application module has a dedicated brand colour, sourced from the
`core.app_modules` database table.

### CSS Variables

```css
--module-core: 239 102 241;
--module-security: 139 92 246;
--module-audit: 100 116 139;
--module-hr: 16 185 129;
--module-payroll: 245 158 11;
--module-benefits: 236 72 153;
--module-talent: 6 182 212;
--module-learning: 20 184 166;
--module-recruitment: 249 115 22;
```

### TypeScript

```ts
import { moduleColors, getModuleColor } from "@afenda/ui/tokens";

moduleColors.hr;              // "#10b981"
getModuleColor("recruitment"); // "#f97316"
```

## Semantic Tokens

Semantic tokens adapt to light and dark mode via the `.dark` class on `<html>`.

| Token | Usage |
|-------|-------|
| `--background` | Page background |
| `--foreground` | Default text |
| `--card` / `--card-foreground` | Card surfaces |
| `--popover` / `--popover-foreground` | Dropdown/popover surfaces |
| `--primary` / `--primary-foreground` | Primary actions, links |
| `--secondary` / `--secondary-foreground` | Secondary buttons |
| `--muted` / `--muted-foreground` | Subtle backgrounds, helper text |
| `--accent` / `--accent-foreground` | Hover/focus highlights |
| `--destructive` / `--destructive-foreground` | Error, danger actions |
| `--border` | Default border colour |
| `--input` | Input border colour |
| `--ring` | Focus ring colour |

### Usage in Tailwind

```html
<div class="bg-background text-foreground">
  <button class="bg-primary text-primary-foreground">Save</button>
  <span class="text-muted-foreground">Helper text</span>
</div>
```

## Border Radius

| Token | Value |
|-------|-------|
| `--radius` | `0.5rem` |
| Tailwind `rounded-lg` | `var(--radius)` |
| Tailwind `rounded-md` | `calc(var(--radius) - 2px)` |
| Tailwind `rounded-sm` | `calc(var(--radius) - 4px)` |
